1. 개요

- 정직하게 이분탐색을 해라라고 주어지는 문제가 아니라면 parametric search일 확률이 100%다
- 특정 값을 x 로 설정했을 때 N 이상인가 아닌가?



2. 이 문제가 이분탐색인지 어떻게 아는가

- 우선 최적화 문제여야한다. 최소값, 최대값 구하기 등등
- 최소 혹은 최대 얘기가 있고, 값이 무지막지하게 크거나 뭔가 logn으로 떨어지는 것이 있을 것 같을때
- 그리디, dp가 적용될 수 없어야한다.
- 이분탐색 틱 해야한다. (많은 문제 풀이 필요)



3. 어떻게 푸는가

* 최적화 문제를 결정 문제로 바꾸는 것이 매우 중요하다.

- 마구간 문제, 공유기 설치
* 최적화 문제 : 말 c마리를 일직선에 배치했을 때 가장 가까운 두 말의 거리가 최대가 되게끔
* 결정 문제 : 두 말의 최대 거리를 x로 설정했을 때 말 c마리 이상인가 아닌가?


- 뮤직 비디오 문제
* 최적화 문제 : DVD가 m개 있을 때 모든 곡을 담을 수 있도록 DVD의 최소 용량 설정
* 결정 문제 : DVD의 용량을 x로 설정했을 때 모든 곡을 담으면 DVD가 m개 이상인가 아닌가?

***** 결정 문제로 얻은 함수가 감소 또는 증가 함수여야한다.







***** 이분 탐색 뿐 아니라 helper method도 중요

-> 1. 일단 이분탐색의 범위는 문제에서 요구하는 값의 범위로 설정.
-> 2. while 문에서 도출된 capa값(mid)을 헬퍼메서드에서 야무지게 요리하여 답을 도출해내야함(보통 return cnt or sum 으로 while 문 안에 박는다). capa는 이분탐색 범위 값중 하나이며, 나중에 m값과 비교를 통해 capa를 조절해야함.
